## Bugs / issues found during dogfood

1. **Critical: 30-min budget threshold bug in `prioritize.ts`** — The `onlyDocs` filter used
   `remaining < 30 * 60_000` which always fired on a 30-minute budget, returning 0 candidates. Fixed to
   `5 * 60_000` (5 min reserve). Same bug in `pipeline.ts` line 87.

2. **Windows spawn ENOENT** — `claude` binary needs `cmd /c claude.cmd` on Windows. `resolveSpawn()`
   existed but only handled explicit `.cmd` extension; added `where claude.cmd` probe for bare names.

3. **stream-json requires `--verbose`** — `claude -p --output-format stream-json` fails with
   "stream-json requires --verbose". Fixed by adding `--verbose` to claude args.

4. **Prompt passed as CLI arg** — On Windows, command lines >8191 chars are truncated by cmd.exe.
   Prompt must be passed via stdin (write to child.stdin, then close). Fixed in executor.ts.

5. **Template rendering: `todo_lines` as `[object Object]`** — The `join_lines` filter called
   `String()` on `{line, text}` objects. Fixed to render `line N: <text>`.

6. **Stale `--include-partial-messages` flag** — Passed but not validated to cause visible issues;
   left in place (harmless).

7. **False positives dominate the candidate set** — All 25 candidates were `research-dossier` type
   from git-grep TODO hits. The hits were overwhelmingly in docs, design specs, test fixtures, and
   the executor's own title-builder string. Only a handful (~5) were in real source files with
   meaningful action items. No `fetch-docs` or PR candidates generated.

8. **11 tasks from attempt 1 produced empty OUT.md** — The `extractLastAssistantText` fallback
   parsed stream-json format (requiring `--verbose`) but the first run used plain text mode. The
   content was generated but not captured. Dedup correctly prevented re-running those tasks.

## What to fix in v0.2

- **Scanner false-positive filter**: Skip lines where TODO is inside a string literal, template
  expression, or inside `*.md` / `*.test.*` files unless the match is on a `//` comment line.
- **Re-run failed first-attempt tasks**: The 11 tasks from attempt 1 produced content (visible in
  the jsonl logs) but empty OUT.md. A `--force-rerun` flag or a 0-byte-OUT.md repair pass would
  recover this content.
- **JSONL discovery**: The session JSONL discoverer found 1 session candidate; the git-grep TODO
  discoverer found 24. Most real value would come from JSONL (in-progress work) not docs TODOs.
  The `discoverJsonl` source only picks sessions where the `ai_title` matches TODO_TITLE_RE — too
  narrow. Consider broader discovery (all recent sessions, not just TODO-titled ones).
- **`dep` candidate discovery**: `discoverDeps` found 0 candidates. Need to confirm it scans
  `package.json` dependencies and emits `fetch-docs` candidates. If broken, that's a v0.2 fix.
- **Slug collision when multiple TODOs in one file**: Two TODOs in `foo.ts` both slug to the same
  dir name. Fix: include line number in slug.
- **Timer leak**: `setTimeout` in executor.ts is never cleared on early task exit. Low priority but
  should be fixed.
- **Candidate quality signal**: Add a "skip docs/fixtures/tests" filter option to `discoverGit`.
  Alternatively, weight down candidates where the file path matches `docs/**`, `test/**`, `*.test.*`.
