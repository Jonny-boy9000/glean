## Key Interactions to Verify
- Run hits a 5-hour session limit → sleeps (interruptibly) → resumes and finishes.
- Run hits a weekly limit → stops, reports "drained weekly capacity".
- Run hits an ambiguous/unclassifiable signal → fail-safe STOP.
- `glean stop` during a drain sleep → run exits within one tick (~30-60s).
- draft-impl task → worktree on the configured base_branch, commit on prep/glean-*, main HEAD untouched.
- `glean morning` → branch entry shows diff stat, the (correct) review command, discard command, trust line.

## Edge Cases
- Spawned session edits but never commits → executor auto-commits the dirty worktree.
- Spawned session produces nothing → status failed, worktree kept for inspection.
- base_branch missing from config → draft-impl skipped with a warning, run continues.
- Resume cursor file missing/corrupt → fresh start, no crash.
- N consecutive unproductive sleeps → circuit-breaker hard stop.
- Anti-spill margin: reset timestamp within margin → refuse to start a new task.
- Spawned draft-impl session attempts `git switch main` / `git reset` → blocked by hardened deny-list AND worktree isolation; main HEAD provably unchanged.
- Non-interactive Bash: spawned session can run `git commit` + test command without hanging on an approval prompt (THE SPIKE — gate v0.7.1 on this).

## Critical Paths (regressions — must not break)
- Weekly/ambiguous signal still STOPS (old behavior preserved while session now sleeps).
- Existing dossier INDEX rendering unchanged for file-type entries.
- `main` checkout never mutated by any draft-impl run.
