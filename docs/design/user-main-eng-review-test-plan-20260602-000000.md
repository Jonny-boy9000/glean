## Affected Surfaces (CLI subcommands)
- `glean run --drain` (NEW flag) — guarded burst; the heart of the drain loop.
- `glean schedule enable|disable|status` (NEW subcommand) — Windows Task Scheduler XML.
- `glean morning` — now aggregates ALL bursts in a drain window, not just the latest run.
- `glean stop` — now timestamp-scoped: stops the current window, not every future weekend.
- `glean run` (bare, no --drain) — MUST stay byte-identical (regression guard).

## Key Interactions to Verify
- Wake before next_eligible_at → exits 0, writes NOTHING (no run row, no gc/sweep/discovery).
- Wake after next_eligible_at → resumes; completed task_ids are skipped, not re-run.
- Burst hits 5-hour (session) limit → persists next_eligible_at + completed set, exits 0 (does NOT abandon the run).
- Burst hits weekly limit → marks week_exhausted, stops, reports "drained weekly capacity".
- Ambiguous/unparseable rate-limit signal (2nd consecutive) → fail-safe stop, never an unbounded re-entry loop.
- 3 consecutive zero-task bursts → no-progress backstop stops the window.
- `glean stop` mid-weekend → next heartbeat's guard exits; next Thursday's fresh window runs (stale STOP ignored by timestamp).
- `glean morning` after a multi-burst weekend → ONE receipt aggregating all branches + dossiers across the window.
- Laptop asleep all weekend (0 bursts) → receipt says "0 bursts", NOT "drained weekly capacity".
- `glean schedule enable` → a queryable Windows task whose XML contains StartWhenAvailable=true, DisallowStartIfOnBatteries=false, StopIfGoingOnBatteries=false, MultipleInstancesPolicy=IgnoreNew, and the repetition interval/duration.

## Edge Cases
- budget.json missing → fresh window (legit first run). Present-but-corrupt → fail-safe exit, NOT a fresh window (anti-spill).
- Concurrent heartbeat reads budget.json mid-write → atomic temp+rename means it sees old or new, never torn.
- Stale RUN.lock after reboot/PID-recycle (>20 min, no fresh summary.json) → reclaimed, not a permanent "busy" no-op.
- Weekly signal with unparseable reset timestamp → window still clears at next scheduled trigger via the duration backstop.
- DST transition mid-weekend → UTC-internal timestamps, no off-by-one in the next_eligible_at compare.
- Discovery (`gh`/network) fails on a re-entry → skip the burst, exit 0, retry next heartbeat (NOT a crash).

## Critical Paths (must not break)
- Bare `glean run` (no --drain) path unchanged — same summary, same exit codes (D5→B keeps the burst pure).
- `main` checkout never mutated by any drain burst (inherited draft-impl invariant).
- Weekly/ambiguous signal still STOPS (old stop-on-rate-limit behavior preserved; only session now pauses/resumes).
- Existing dossier INDEX rendering unchanged for file-type entries.
