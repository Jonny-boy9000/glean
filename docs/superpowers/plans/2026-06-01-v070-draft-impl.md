# v0.7.0 draft-impl Engine Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `draft-impl` candidate type that provisions an isolated git worktree on a `prep/glean-<taskid>` branch off the project's configured `base_branch`, spawns `claude -p` (deny-list + Bash allowed) to implement the top TODO, auto-commits, and captures a `git diff --stat` for a future receipt — never touching the user's main checkout.

**Architecture:** Pure refactors land first (candidate-meta extraction, CandidateType alias in memory, centralized deny-list) so the engine builds on DRY foundations. Then the `TaskResult.output` discriminated union (`file` | `branch`) is introduced before the engine and index renderer consume it. The engine reuses the existing `executeOne` spawn plumbing, branching on `c.type === 'draft-impl'`. Memory schema migrates to v4 to persist diff-stat columns.

**Tech Stack:** Node 20 + TypeScript, vitest, better-sqlite3, citty, git/gh shell-out, Windows-first (jobobject taskkill /T).
